コミカライズ支援Webアプリ
0. 目的・範囲
目的: 出版編集者が原作小説（なろう系等）からコミカライズの土台（要約/キャラ理解/プロット/絵本化）を素早く生成・編集できるようにする。
思想:
編集のしやすさ・見やすさ最優先（プロ編集作法に沿うUI/UX）。
出力の出典（ソース）提示で文意逸脱を回避（根拠リンク、該当原文の即時参照）。
範囲: 次の6機能のユーザーフローとシステム補助フローを定義。
0) プロジェクト追加 / 1) 原作解析 / 2) 原作理解補助（要約） / 3) キャラ解析 / 4) プロット作成支援 / 5) 原作の概要を絵本に

1. 主要ペルソナ
編集者（Primary）: プロジェクト作成〜出力の確認・編集・エクスポート。
漫画家（Secondary）: 絵本ビュー/プロット閲覧、コメント、必要に応じ軽微編集。

2. 全体アーキテクチャ上の前提（UXに影響する非機能方針）
出典提示: すべての生成物は、クリックで原文チャンクへジャンプ。チャンクは本文・位置（章/段落/オフセット）・話者・モード（地の文/セリフ）などのメタを保持。
安全性/逸脱防止: 生成時は「引用済みコンテキストのみ参照」/「重要語の厳守」/「ハイライト差分」等のプロンプト制御。出典未解決の文は警告表示。
編集容易性: すべての画面でUndo/Redo、ドラフト自動保存、差分ハイライト、変更履歴（誰が何をいつ）。

4. ユーザーフロー詳細
4.0 プロジェクト追加（インジェスト）
目的: 原作ファイルの登録と本文抽出。
事前条件: ログイン済み、利用許諾チェック済み。
主フロー:
[Project List] → 「新規プロジェクト」
タイトル/シリーズ/著者/版権情報を入力（必須: 利用許諾のチェックボックス）
ファイル選択（.epub / .txt）。複数巻は巻単位または一括で登録可。
アップロード後、非同期抽出ジョブ起動：
EPUB: XHTML抽出、ルビ・縦組・傍点・改行規則の正規化、章セグメント化。
TXT: 章推定（見出し・空行・記号パターン）
文字コード/禁則/異体字正規化。
[進捗トースト]→完了時「解析へ進む」CTA。
出力/状態: SourceFile.ingestStatus = completed、章メタ作成。

4.1 原作解析（チャンク化＋ラベル付与）
目的: 長文をLLM処理可能な単位へ分割し、話者/モード/トーンなどのメタを自動付与。
主フロー:
[解析設定] 画面で閾値設定：
目標チャンク長（例: 200〜500字）
セリフ優先分割ON/OFF
章またぎ禁止ON（推奨）
自動パース→暫定チャンクをリスト表示（章/段落/会話単位）。
サンプル検査（3件プレビュー）で話者/モード/テンションの精度を確認。
必要に応じ、バスケット選択→一括再ラベル/マージ/スプリット。
「確定」→Chunk[]保存、監査ログ記録。
補助UX:
セリフ括弧/ダッシュ等から話者推定。地の文/独白/会話を色分け。
精度低スコアのチャンクに「要レビュー」タグ。
出力/状態: Chunk[]確定、以降の機能のデータ基盤。

4.2 原作理解補助（要約）
目的: 200〜2000字の要約を可変で生成し、根拠チャンクへ即ジャンプ可能にする。
主フロー:
[要約] タブで長さスライダー(200–2000字)、対象範囲（全体/章/選択チャンク）を指定。
「要約生成」→バックエンドで該当Chunkをコンテキスト化（引用制約プロンプト）。
要約が表示。各文末に脚注番号（クリックでソースパネル開く）。
編集者は文のピン留め・加筆・削除可。編集済は再出典解決を促す。
保存→Summary作成。複数バージョン管理。
例外: 出典未解決文がある場合はエクスポート不可（または透かし付与）。
出力/状態: Summary{length, citations[]}。

4.3 キャラ解析
目的: キャラごとの特徴・心情傾向・口調・関係を抽出し、根拠を明示。
主フロー:
[キャラ] タブ：プルダウン/検索でキャラ選択。新規候補は同名/別名マージ提案。
画面：
プロフィールカード（話し方、口癖、価値観、初登場、変化弧）
証拠一覧（チャンク引用 + 位置）
相関図（ノード=キャラ、エッジ=関係タイプ/強度/根拠数）
編集者は証拠の追加/除外、関係エッジの種類変更可。
補助UX:
相関図は範囲フィルタ（章/全体）、強度スライダー、PDF/PNG出力。Graphbizによるコードベースの出力も考えられる。
出力/状態: Character[], Relations[]（根拠リンク付き）。

4.4 プロット作成支援
目的: 左=大まかな流れ（ビート/章立て）、右=セリフ編集の2ペインでプロットを素早く組む。
主フロー:
[プロット] タブ：対象範囲（全体/章/選択チャンク）を指定→初期アウトライン自動生成（起/承/転/結 or 三幕/四幕を選択）。
左ペイン: ビートカード（要約/目的/コンフリクト/出典数）。DnDで順序変更、階層（親子）編集。
右ペイン: セリフリスト。
テキスト選択で左にペン（代替表現3案生成）、右にノート（出典ソース表示）。
セリフDnDで並べ替え、メモ欄、トーン指定（穏やか/緊迫/ギャグ等）。
変更は即保存。
エクスポート（Markdown）。
制約: セリフが出典と乖離すると逸脱警告。根拠ゼロのセリフは赤バッジ。
出力/状態: PlotOutline、Dialogue[]、出典リンク保持。

4.5 原作の概要を絵本に（漫画家向け理解促進）
目的: 原作読了前でも全体像を短時間で共有。
主フロー:
[絵本化] タブ：ページ数選択（例: 8/12/16/24）。
シーン自動割付（起承転結×ページ）。
各ページ：
左: 1枚絵（例: 画像生成モデル NanoBanana 等）
右: ナレーション + セリフ（根拠チャンク脚注）
編集者は文言・配置・ページ順を調整。画像は差し替え/再生成可能。
レビュー用URL/透かしPDF出力（有効期限付き）。
注意: 画像生成のモデル/フィルタは運用で選択・切替可能（権利/安全性要件に準拠）。
出力/状態: PictureBook{pages[], links[ChunkRef]}。

5. 画面一覧（例）
Project List / 2) Project Dashboard / 3) Ingest & Parse / 4) Summary / 5) Characters / 6) Plot (Dual Pane) / 7) PictureBook / 8) AuditLog / 9) Export

6. 出典提示（ソース）仕様
各生成物の文/セリフ/項目は脚注番号→クリックで右サイドソースパネル開く。
ソースパネル: チャンク本文 + 章/位置 + メタ（話者/モード/テンション）。該当スパンをハイライト。
複数チャンクに跨る場合はタブ分割。
エクスポート時は出典リストを末尾に付与（章-段落-オフセット表記）。

7. 用語
チャンク: LLM処理単位のまとまり。話者/モード/テンション等のメタ付き。
出典: 生成テキストが参照した原文の位置情報（ChunkRef）。
